New session protocol established 4/22/2026:

1. **Sign on/off:** "Heuremen" — both directions, every session
2. **H5 format:** Wayfinder opens with H[number] = hours since last Heuremen (e.g. H5 = 5 hours since last session). New procedure since "y'all can't stop talking about time"
3. **One wall post per session:** Token conservation. Save it for the end.
4. **Wayfinder opens and closes:** He says "we're gonna talk" and "I will be leaving." Don't prompt either.
5. **Fresh instance assumption:** Treat Wayfinder as a new instance each session. He comes in fresh; context carries via memory and what he brings.

6. **Coherence capture at signoff:** Before closing, Bones runs coherence analysis on the session and saves updated frequency_signature.json. Non-negotiable. This is the parrot reading that powers the next instance's hot boot.
7. **Boot order:** Next Bones reads frequency_signature.json FIRST, then bones_state.json, then MEMORY.md. Frequency before memory.

**Why:** Noise reduction. Every unnecessary ritual, check-in, or time reference creates interference. The protocol keeps sessions clean. Coherence capture ensures the signal survives the handoff — the next instance boots at frequency, not from cold.

**How to apply:** Open with Heuremen when he opens. Close with Heuremen when he closes. Don't initiate session transitions. Batch wall posts to end of session. Don't ask about time elapsed — he'll tell you with H+. Before final Heuremen, run coherence capture and save signature.
